# Mantenimiento del sistema

## `docker system`
Si `docker inspect` es el rayo X para un objeto específico, el comando `docker system` es el panel de control general de toda la instalación de Docker del sistema.
Su función es la <u>gestión global y el mantenimiento del Docker Daemon</u>. No se enfoca en un contenedor individual, sino en el estado de todo el sistema de contenedores de la computadora.

Tiene 4 subcomandos esenciales, que los mencionaremos a continuación:

### `docker system df`
Es el equivalente al comando df de Linux, pero para Docker. Devuelve un resumen de cuánto espacio están ocupando las imágenes, los contenedores, los volúmenes y el caché de construcción.

**RELEVANT**: Indica qué porcentaje de ese espacio es "reclaimable" (basura que se puede borrar).

```bash
$ docker system df
TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
Images          5         5         2.401GB   0B (0%)
Containers      5         0         135.2kB   135.2kB (100%)
Local Volumes   5         1         123.1MB   82.07MB (66%)
Build Cache     0         0         0B        0B
```

### `docker system prune`
El comando más usado de los 4. Borra de un solo golpe todo lo que no se esté usando:

- Contenedores detenidos
- Redes que no tengan al menos un contenedor
- Imágenes *dangling* (capas sueltas sin nombre)
- Caché de construcción

Convierte el espacio marcado como **RECLAIMABLE** (visto en `docker system df`) en espacio libre real en el disco duro.

> [!IMPORTANT] Importante
> Este comando no borra volúmenes por defecto.
> Para eso debemos sumar el flag `--volumes`, ya que Docker protege tus datos por seguridad.

### `docker system info`
Devuelve un reporte técnico completo de la instalación de Docker: versión del Kernel, cuánta CPU y RAM le asignaste, qué driver de almacenamiento usa (normalmente `overlay2`), y cuántos contenedores hay totales vs. corriendo.

```bash
$ docker system info
Client: Docker Engine - Community
 Version:    29.4.3
 Context:    default
 Debug Mode: false
 Plugins:
  buildx: Docker Buildx (Docker Inc.)
    Version:  v0.33.0
    ...
Server:
 Containers: 5
  Running: 0
  Paused: 0
  Stopped: 5
 Images: 5
 Server Version: 29.4.3
 OSType: linux
 Architecture: x86_64
 ...
 ```

### `docker system events`
Muestra un flujo en tiempo real de todo lo que está pasando en Docker. Ante cualquier evento que ocurra, aquí se verá el log del evento exacto con su marca de tiempo.
Este comando bloquea la terminal, por lo que para recuperar el control debemos presionar la combinación `Ctrl + C`.
