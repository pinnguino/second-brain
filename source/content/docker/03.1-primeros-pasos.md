# Primeros pasos

## Nuestro primer contenedor
Para comprobar que hemos instalado Docker de forma correcta, corramos una imagen.
Docker, en sus instrucciones de instalación nos sugiere probar la imagen de prueba llamada `hello-world`, así que prosigamos con esa.

```bash
$ docker run hello-world
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
4f55086f7dd0: Pull complete 
d5e71e642bf5: Download complete 
Digest: sha256:f9078146db2e05e794366b1bfe584a14ea6317f44027d10ef7dad65279026885
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
    (amd64)
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it to your terminal. To try something more ambitious, you can run an Ubuntu container with: $ docker run -it ubuntu bash Share images, automate workflows, and more with a free Docker ID: https://hub.docker.com/

For more examples and ideas, visit:
 https://docs.docker.com/get-started/
```

Si obtenemos el mensaje `Hello from Docker!`, vas por buen camino!

Notar que en una instalación nueva de Docker, no tendremos ninguna imagen descargada en nuestro sistema. Por eso, al correr la imagen `hello-world`, docker notifica en la primera línea que no encontró la imagen de forma local, por lo que la descarga de [Docker Hub](https://hub.docker.com/_/hello-world). Este es el comportamiento predefinido del comando `run`.

## Ver las imágenes y contenedores

### Imágenes
Con el comando `docker images` podemos ver las imágenes que están instaladas de forma local en nuestro sistema.

```bash
$ docker images
                                                                                                                                                         i Info →   U  In Use
IMAGE                ID             DISK USAGE   CONTENT SIZE   EXTRA
hello-world:latest   f9078146db2e       25.9kB         9.49kB    U   
```

### Contenedores
Con el comando `docker ps` podemos listar los contenedores. El comando sin argumentos muestra sólo los contenedores que están corriendo al momento de ejecutar el comando.

```bash
$ docker ps
CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES

```

En este caso, está vacío porque el contenedor que ejecutamos antes, muestra los mensajes que se vieron anteriormente y se detiene. En el caso de que querramos **ver todos los contenedores** (incluso los que no están corriendo), podemos hacerlo con `docker ps -a`:

```bash
$ docker ps -a
CONTAINER ID   IMAGE         COMMAND    CREATED          STATUS                      PORTS     NAMES
d47d2f3dd55d   hello-world   "/hello"   23 minutes ago   Exited (0) 23 minutes ago             epic_payne
```

# Docker Hub
[Docker Hub](https://hub.docker.com/explore) es el mayor registro y repositorio público en la nube de imágenes de contenedores Docker. Funciona como una biblioteca centralizada donde desarrolladores y equipos pueden buscar, descargar (pull), subir (push) y compartir imágenes preconfiguradas, facilitando la automatización, colaboración y el despliegue de aplicaciones en contenedores.
Cuando hicimos el comando `docker run hello-world`, Docker hizo un `pull` desde este repositorio.

## Características clave de Docker Hub
**Biblioteca de Imágenes Oficiales**: Ofrece miles de imágenes certificadas y mantenidas por Docker y grandes proveedores de software (por ejemplo, imágenes oficiales de Python, Nginx, MySQL).
**Repositorios Públicos y Privados**: Permite almacenar imágenes públicas para la comunidad o privadas para uso interno de equipos de trabajo, ofreciendo control de versiones.
Automatización (Builds): Posibilita conectar repositorios de código fuente (como GitHub o Bitbucket) para crear automáticamente imágenes cuando se actualiza el código.

## Descargando una imagen
Descarguemos una imagen de Docker Hub y corramosla en nuestra instalación de Docker.
Para esto comenzamos buscando una imagen en el sitio de Docker Hub.

> [!IMPORTANT] Importante
> No todas las imágenes que estén subidas a Docker Hub son gratuitas.
> Las licencias funcionan de la misma forma que las aplicaciones nativas tradicionales.

Para este ejemplo usaremos la imagen de [Ubuntu](https://hub.docker.com/_/ubuntu), una imagen muy usada como "*lienzo en blanco*" para crear entornos de desarrollo personalizados desde cero, probar scripts de shell en entornos seguros, etc.
En ese sitio veremos información sobre la imagen, las arquitecturas sobre la que es compatible, versiones, documentación, licencia, etc.
Para descargar la imagen, veremos que hay un recuadro que muestra los tags, el tamaño y la última vez que se actualizó la imagen, junto con un comando para poder instalarlo.
En este caso, sería `docker pull ubuntu`.

```bash
$ docker pull ubuntu
Using default tag: latest
latest: Pulling from library/ubuntu
b40150c1c271: Pull complete 
92842f25412d: Download complete 
Digest: sha256:c4a8d5503dfb2a3eb8ab5f807da5bc69a85730fb49b5cfca2330194ebcc41c7b
Status: Downloaded newer image for ubuntu:latest
docker.io/library/ubuntu:latest
```

Notar que al hacer un `pull` de la imagen sin tag, Docker automáticamente descarga la imagen con el tag `latest`.
Usar el tag por defecto es cómodo para pruebas rápidas, pero en proyectos serios se suele evitar por estas razones:

- Falta de predictibilidad: Si haces un pull de Ubuntu hoy, `latest` podría ser la versión 22.04. Si lo haces en seis meses, latest podría ser la 24.04. Esto puede romper tu aplicación porque el entorno cambió sin que tú lo decidieras.

- Inconsistencia entre entornos: Tu compañero de equipo podría tener una versión de latest en su máquina y tú otra en la tuya, lo que causa errores difíciles de rastrear.
Para pruebas rápidas no hay problema, pero tener en cuenta que al hacer un pull en 6 meses, la versión `latest` (ya sea que marquemos ese tag intencionadamente o no) podría cambiar y perderíamos la consistencia del entorno a largo plazo.

Luego de descargar la imagen, podemos correr la imagen con el comando `docker run <imagen>`.
En este caso, como la imagen ya se encuentra descargada localmente en nuestro sistema (porque ya hicimos un `pull`), por lo que ahora directamente se creará el contenedor. Que podremos visualizar cuando hacemos el comando `docker ps -a`.

> [!WARNING] Cuidado
> Notar que cuando ejecutamos `docker run`, Docker automáticamente le genera un nombre aleatorio a nuestros contenedores.
> Esto para pruebas iniciales puede estar bien, pero en ambientes de desarrollo productivos no es muy útil.
> Para poder asignarle un nombre a nuestros contenedores debemos agregarle la opcion `--name <tu-nombre>`.
> Para cambiarle el nombre a un contenedor ya creado lo podemos hacer con `docker rename <nombre_actual_o_id> <nuevo_nombre>`.

```bash
$ docker run ubuntu
$ docker ps -a
CONTAINER ID   IMAGE         COMMAND       CREATED         STATUS                     PORTS     NAMES
227d70dc5006   ubuntu        "/bin/bash"   7 seconds ago   Exited (0) 6 seconds ago             inspiring_noether
d47d2f3dd55d   hello-world   "/hello"      3 days ago      Exited (0) 3 days ago                epic_payne
```

> [!IMPORTANT] Importante
> Notar que:
> Si ejecutamos `docker run <imagen>` varias veces con una misma imagen, crearemos varios contenedores de la misma imagen.

# El comando `docker ps`
Como ya hemos visto anteriormente, este comando nos permite obtener información sobre los contenedores que están creados en el sistema.
Podemos ver la ayuda de las posibles opciones con el comando `docker ps --help`.

```bash
$ docker ps --help
Usage:  docker ps [OPTIONS]

List containers

Aliases:
  docker container ls, docker container list, docker container ps, docker ps

Options:
  -a, --all             Show all containers (default shows just running)
  -f, --filter filter   Filter output based on conditions provided
      --format string   Format output using a custom template:
                        'table':            Print output in table format with column headers (default)
                        'table TEMPLATE':   Print output in table format using the given Go template
                        'json':             Print in JSON format
                        'TEMPLATE':         Print output using the given Go template.
                        Refer to https://docs.docker.com/go/formatting/ for more information about formatting output with templates
  -n, --last int        Show n last created containers (includes all states) (default -1)
  -l, --latest          Show the latest created container (includes all states)
      --no-trunc        Don't truncate output
  -q, --quiet           Only display container IDs
  -s, --size            Display total file sizes
```

Las opciones más útiles pueden ser `-a`, `-l`, y `-f`. Se puede filtrar/formatear el listado de contenedores en base al nombre, red, volumen, id, entre muchas otras opciones. Para más información referirse a la [documentación oficial de Docker](https://docs.docker.com/reference/cli/docker/container/ls).
