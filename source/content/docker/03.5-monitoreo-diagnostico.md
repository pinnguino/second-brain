# Monitoreo y diagnóstico

## `docker logs`
Supongamos que tenemos un contenedor que está corriendo en modo detached (Background). Cuando esto sucede no vemos qué sucede en ese contenedor. Una de las formas de visualizar lo que está pasando en el mismo sin intervención del usuario es con el comando `docker logs`.
Esto muestra la salidas del contenedor en nuestra sesión de terminal, incluso funciona si el contenedor está apagado. Un ejemplo de uso sería ver las conexiones que se realizaron en una base de datos mirando los logs del cliente `psql`.

Opciones comunes para `docker logs`:

- `--tail N`: Con esta opción le decimos al daemon que nos muestre los últimos `N` logs del sistema.
- `-f`: La terminal se queda imprimiendo los logs del contenedor de forma indefinida.

## `docker top`
El comando `docker top <ID_CONTENEDOR>` <u>muestra los procesos en ejecución dentro de un contenedor</u>. Permite visualizar en tiempo real qué comandos, usuarios (UID) y IDs de proceso (PID/PPID) están activos, siendo ideal para auditorías rápidas y depuración sin entrar en el contenedor. También sirve con el nombre del contenedor.

```bash
$ docker top mi-contenedor # O el ID del contenedor
UID                 PID                 PPID                C                   STIME               TTY                 TIME                CMD
root                8953                8930                0                   18:51               pts/0               00:00:00            /bin/bash
root                9084                8953                27                  18:51               pts/0               00:01:15            yes
```

## `docker stats`
Este comando, invocado con `docker stats <ID_CONTENEDOR>` (también funciona con el nombre del contenedor) tiene como función mostrar datos en tiempo real de los recursos que está utilizando el contenedor. Este comando bloquea la terminal, de forma que para salir debemos presionar la combinación `Ctrl + C` para recuperar el control. Podemos ver el porcentaje de uso de la CPU, memoria RAM, si está enviando o recibiendo datos de la red, entre otras cosas. 

```bash
$ docker stats mi-contenedor # O el ID del contenedor
CONTAINER ID   NAME      CPU %     MEM USAGE / LIMIT     MEM %     NET I/O          BLOCK I/O      PIDS
29b71f3fe30e   ubuntu    18.38%    1.16MiB / 15.31GiB    0.01%     4.17kB / 126B    53.2kB / 0B    2
```

## `docker inspect`
Este comando es, en esencia, el *rayo X* de Docker. Mientras que comandos como `docker ps` devuelven un resumen rápido, `docker inspect` <u>devuelve toda la metadata de un objeto en un formato JSON detallado</u>.
Sirve para ver la configuración interna de contenedores, imágenes, redes o volúmenes.

La salida de este comando es un JSON muy grande, por lo que ejecutarlo directamente en una terminal sería inmanejable. Lo más recomendable es **redirigir la salida a un archivo**, o **formatear/filtrar la salida** para obtener sólo los datos que nos interesan de la forma que querramos.
Se puede encontrar información sobre el formateo de la salida en la documentación oficial de Docker para el comando `inspect` [aquí](https://docs.docker.com/reference/cli/docker/inspect).
